## Что такое вехи?

Вехи — это способ организации задач в Foundation. Они могут использоваться для группировки задач на основе общей темы, проекта или временной шкалы. Например, вы можете создать веху для конкретного релиза функции, спринта или фазы проекта. Вехи можно использовать для отслеживания прогресса, установки дедлайнов и обеспечения того, чтобы задачи выполнялись в логическом порядке.

> **Примечание:** Вехи специфичны для проекта и не могут быть перемещены в другой проект в Трекере.

## Создание новой вехи

Перейдите в Трекер и выберите Вехи под любым проектом. Затем нажмите кнопку `+ Веха` в правом верхнем углу. Здесь вы можете ввести название и необязательное описание для вашей Вехи, а также установить статус и дедлайн.

## Добавление задач к вехе

### Создание новой задачи внутри вехи

Есть несколько способов добавить задачи к вехе. Если вы ещё не создали задачу, просто нажмите кнопку `+` на странице деталей любой вехи. Это позволит вам создать новую задачу, которая будет автоматически связана с этой вехой.

В качестве альтернативы, при создании новой задачи из любого другого места в трекере, просто выберите желаемую веху из выпадающего меню в диалоговом окне создания задачи:

Если вы уже создали задачу и хотите связать её с существующей вехой, вы можете открыть страницу деталей этой задачи и выбрать желаемую веху в правой боковой панели:

## Использование вех для сортировки задач

Как и любой другой атрибут задачи, вехи можно использовать для фильтрации и сортировки задач в Трекере. Например, вы можете переключать, хотите ли вы видеть веху для каждой задачи в вашем Трекере, или вы можете настроить весь вид Трекера, организовав задачи по вехам.

Задачи также можно фильтровать по вехам, что позволяет быстро увидеть все задачи, связанные с выбранными вехами.
